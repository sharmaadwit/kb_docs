# Journey Canvas Controls

**Module**: Bot Studio

## Definition
This section provides tools and features to effectively design, manage, and optimize your journeys on the canvas. Explore options like Block Templates, API Management, Variable Configuration, and other essential controls that streamline the journey-building process.

## Procedure
### Exact UI path
Gupshup Console → Bot Studio → Journey Canvas Controls

### Prerequisites
- Access to the relevant bot/project in Gupshup Console.
- A journey/app where you can test the configuration.

### Fields to configure
- No explicit fields were identified in the source; use the UI controls shown on this page.

### Steps
1. Open Gupshup Console.
2. Go to **Bot Studio**.
3. Go to **Journey Canvas Controls**.
4. Click **Save** (or **Save & Deploy**) to apply changes.

### Validation / where to check
- Run the flow in **Test your Bot** and confirm the expected node/path executes.
- If the change must affect live traffic, use **Save & Deploy** and verify on the target channel.

### Troubleshooting
- If behavior is unchanged, confirm you updated the correct node and used **Save & Deploy** for live channels.
- If the wrong branch/path runs, re-check conditions, connected nodes, and fallback connectors.

### Save / publish / deploy behavior
- Click **Save** (or **Save & Deploy**) to apply changes.

### Setup path
- Go to **Bot Studio**.
- Go to **Journey Canvas Controls**.
